About the README

This sample file is pre-loaded into the sample text editor so that you can see Fragaria working right away. It also discusses some of the sample techniques used in the demo application in order to take advantage of some of Fragaria’s key features.

About MGSFragariaViewModern demo app

The demonstration application is meant to show you how you might use several of the features available in Fragaria, and in particular MGSFragariaView which is destined to replace MGSFragaria for mainstream use.

The demo application is not meant to be a full-featured application, but only a model for demonstrating how you might interact with Fragaria. For example your own application would have more sophisticated means of managing breakpoints and syntax errors.

This demo application uses purely "new" components that are 100% backwards incompatible with nearly every other Fragaria fork, such as the syntax error controller and the new-style defaults coordinator and display settings panels.

About Fragaria

This fork of Fragaria really reimplements Fragaria from the very lowest levels, but is not a complete rewrite. Some of the most fundamental differences include:

What end users should notice:

  • A much improved syntax highlighter with better speeds.
  • A better, more capable gutter with super-fast response (no more tearing on long documents).
  • Improved syntax error display options.
  • Modern-looking settings panels, which can be used for managing Preferences (for example).

What developers should notice:

  • Fragaria is no longer dependent upon user defaults for controlling its behavior and appearance.
  • Property-based API for controlling behavior and appearance.
  • Support for Fragaria’s legacy defaults is still present, however (optional).
  • A new defaults system that can optionally be used.
  • A better syntax error controller with more features for types and styles of syntax errors.
  • Improved breakpoints features that support multiple colors and styles.

Using the MGSFragaria class is largely backwards compatible. See for example the legacy “NSDocument based” and “Simple” demo application build targets.

Features this demo implements

Split View

Early in applicationDidFinishLaunching we replace the text storage of the lower view with the text storage of the upper view. This effectively makes the two views into a split view.

Syntax errors

The demo application statically adds syntax errors, and shows off the different types of errors that can be used easily. Consult SMLSyntaxError.h for a description of the error properties.
